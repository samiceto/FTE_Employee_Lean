## 🚀 How It Works

### 1. Post Tracking

When a watcher posts content, it automatically tracks:

**Instagram:**
- Caption (first 500 chars)
- Timestamp
- Likes, comments, shares, reach (placeholder for engagement tracking)

**Twitter/X:**
- Text (up to 280 chars)
- Timestamp
- Likes, retweets, replies, views, hashtags

**Facebook:**
- Message (first 500 chars)
- Timestamp
- Likes, comments, shares, reactions

### 2. Daily Summary Generation

Every 24 hours, each watcher:
1. Loads tracked posts from last 24 hours
2. Calls the appropriate analyzer skill (uses Groq LLM - FREE!)
3. Generates comprehensive markdown report
4. Saves to `Summaries/<Platform>/` directory

### 3. LLM-Powered Analysis

Each analyzer skill uses Groq LLM to provide:
- **Performance summary** (2-3 sentence overview)
- **Key insights** (3-5 insights about what's working/not working)
- **Recommendations** (3-5 actionable suggestions to improve engagement)
- **Content analysis** (themes, types, hashtags that perform well)
- **Engagement trends** (patterns in audience interaction)

## 💻 Usage

### Running the Watchers

Each watcher automatically generates summaries while posting:

```bash
# Instagram Watcher
python src/watchers/insta_watcher.py

# Twitter/X Watcher
python src/watchers/x_watcher.py

# Facebook Watcher
python src/watchers/fb_watcher.py
```

### Summary Generation Schedule

- **Automatic:** Checks every hour if 24 hours have passed since last summary
- **First Summary:** Generated when first post is 24+ hours old
- **Subsequent Summaries:** Once per day, analyzing posts from last 24 hours

### Manual Summary Generation

You can also generate summaries programmatically:

```python
from skills import get_registry, SkillInput

# Get analyzer skill
registry = get_registry()
analyzer = registry.get_skill(
    "instagram_analyzer",
    vault_path="/path/to/vault",
    groq_api_key="your_groq_key"
)

# Prepare posts data
posts = [
    {
        "caption": "Check out our new product!",
        "timestamp": "2026-01-21T10:00:00",
        "likes": 150,
        "comments": 25,
        "shares": 10
    },
    # ... more posts
]

# Generate summary
result = analyzer.execute(SkillInput(
    data={
        "posts": posts,
        "period": "daily"
    }
))

if result.success:
    print(f"Summary generated: {result.metadata['report_path']}")
    print(result.result['summary'])
```

## 🔧 Configuration

### Environment Variables

Required in `.env`:
```bash
GROQ_API_KEY=your_groq_api_key_here
```

Get your FREE Groq API key: https://console.groq.com/keys

### Customization

#### Change Summary Frequency

Edit the watcher file:
```python
# In run_watcher() function
if datetime.now() - last_summary_check > timedelta(hours=24):  # Change this
    generate_daily_summary()
```

#### Change Summary Period

When calling analyzer:
```python
skill_input = SkillInput(
    data={
        "posts": recent_posts,
        "period": "weekly"  # Change from "daily" to "weekly"
    }
)
```

## 🐛 Troubleshooting

### No Summaries Generated

**Problem:** Summaries not appearing in Summaries/ folder

**Solutions:**
- Check GROQ_API_KEY is set in .env
- Verify posts exist in PostsTracker/*.json
- Ensure posts are at least 24 hours old
- Check watcher logs for errors

### Empty Summaries

**Problem:** Summary says "No posts to analyze"

**Solutions:**
- Check PostsTracker/*.json has posts from last 24 hours
- Verify timestamp format is ISO format (YYYY-MM-DDTHH:MM:SS)
- Check cutoff date logic in generate_daily_summary()

### LLM Errors

**Problem:** "Groq API error" or similar

**Solutions:**
- Verify API key is valid
- Check Groq API rate limits (free tier)
- Reduce number of posts if hitting token limits
